@keyframes

А что, если нужно сделать много разных действий и в разном порядке?

@keyframes

            @keyframes animationName {
              from { animatedProperty } 
              to { animatedProperty } 
            }

            .selector {
              animation: animationName
              transition: animatedProperty animatedTime timingFunction 
            }

@keyframes

            @keyframes toCheers {
                   from { transform: translateY(-100%); }
                   to { transform: translateY(0); }
            }

            .glass-beer{
                    animation-name: toCheers;
                    animation-iteration-count: infinite;
                    animation-duration: 2s;
            }

@keyframes

animation-name
Определяет имя @keyframes, настраивающего кадры анимации.
animation-duration
Определяет время, в течение которого должен пройти один цикл анимации.
animation-timing-function
Настраивает ускорение анимации.
animation-delay
Настраивает задержку между временем загрузки элемента и временем начала анимации.
animation-iteration-count
Определяет количество повторений анимации; Вы можете использовать значение infinite для бесконечного повторения анимации.
animation-direction
Дает возможность при каждом повторе анимации идти по альтернативному пути, либо сбросить все значения и повторить анимацию.

@keyframes

            @keyframes toCheers {
                   0% { transform: translateY(0%); }
                   50% { transform: translateY(-100%); }
                   100% { transform: translateY(0%); }
            }

            .glass-beer{
                    animation-name: toCheers;
                    animation-iteration-count: infinite;
                    animation-duration: 2s;
            }

Правила анимации:

  1. Transition применяется только к свойствам, у которых могут быть промежуточные значения.
  2. Анимируйте свойства, которые не вызывают перерисовку страницы.
  3. Используйте свойство will-change.

Js/CSS. Проблемы.

  1. При слишком частом вызове события происходит большое количество перерисовок которые тормозят работу страницы.
  2. Если трансформация привязана к событию которое срабатывает слишком часто это может затормозить страницу.

Js/CSS. Решения.

  1. requestAnimationFrame
  2. Throttle 
var start = null;
var element = document.getElementById('SomeElementYouWantToAnimate');
function step(timestamp) {
  if (!start) start = timestamp;
  var progress = timestamp - start;
  element.style.transform = 'translateX(' + Math.min(progress / 10, 200) + 'px)';
  if (progress < 2000) {
    window.requestAnimationFrame(step);
  }
}
window.requestAnimationFrame(step);

Javascript. Canvas

WebGl.

WebGl.

Запускается внутри Canvas и выносит расчеты с CPU на GPU, за счет чего возможна реализация сложных трехмерных анимаций, состоящих из миллионов точек.

WebGl.

Преимущества:

Обрабатывается на GPU, поэтому можно делать очень слоные 3D анимации

Недостатки:

Сложен в освоении. ПРиходится писать код не только на JS, но и на низкоуровневых языках - шейдерах, у которых под капотом C++
